Purpose

An end-to-end payment and operations platform for an international school's cooperative store and canteen. Parents and staff fund campus-card wallets; students and staff spend at POS terminals; admins manage users, departments, refunds, settings, and reports. The system also drives a customer-facing second-monitor display for transparent checkout.

🖥️ Kiosk — Self-Service Touch Terminal

Standalone Vue 3 app deployed separately from the main React app. Designed for unattended touch-screen terminals (typically near canteen / coop entrance) so cardholders can check balance, view history, and top up without queueing at the cashier. No authentication — identification by RFID card tap or manual code entry only. Read-only for sensitive data; write actions go through QR payment, no card storage.

Identification

  • RFID card tap on attached reader (preferred path) High
  • Manual code entry via on-screen Numpad (fallback)
  • Auto-timeout to Welcome screen after 30s idle
  • No password / no PIN — physical card possession is the auth boundary

Balance Check (BalanceView)

  • Display: holder name, current balance, last 3 transactions High
  • Big readable font for touch viewing at arm's length
  • "Top Up" / "Full History" / "Done" action buttons
  • NO photo on screen (privacy — kiosk is in a public area)

  • Paginated list with type icons (purchase / top-up / transfer / refund) Med
  • Date + shop name + amount visible on each line
  • Receipt preview popup (ReceiptPreview component)
  • Filter by date range / transaction type

  • Generate PromptPay QR via backend /wallets/{id}/topup-qr
  • Auto-poll payment status; show success screen on confirmation

Welcome / Idle (WelcomeView)

Operational Constraints

  • Read-only API surface for balance + history (no admin endpoints)
  • Stateless: nothing persisted across sessions — every tap starts fresh
  • Touch-first UI: large hit targets, no tooltips, no hover-only patterns
  • English + Thai i18n switchable from welcome screen
  • Runs in browser fullscreen / kiosk mode (no nav bar)
  • Independent Vercel deployment with its own URL
