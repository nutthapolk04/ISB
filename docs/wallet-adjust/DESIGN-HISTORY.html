5คำแนะนำการ revert

  1. เอา Report section ออก กลับไปเป็นการ์ดเดียว (ย้อนข้อ 2) — ต้องแก้มือ (manual edit) เพราะ logic พันกันแล้ว: แหล่งข้อมูลนักเรียนถูกย้ายไปใช้ /admin/cardholders, backend ของ report ยังต้องอยู่, และ refactor ที่ตามมาทีหลัง (formatTHB / PaginationBar) ต้องเก็บไว้ — ตัดออกได้เฉพาะ JSX ส่วน Report + state/handler ของมันเท่านั้น
  2. กลับไป header ธรรมดา (ย้อนข้อ 1) — แก้จุดเดียวจบ (เปลี่ยน .page-header กลับเป็น h1 ลอย) แต่หน้าจะไม่เข้าชุดกับหน้า admin อื่น ~16 หน้า ที่ใช้ header แบบการ์ดหมดแล้ว
  3. เอา "Credit (+)" / "Debit (−)" กลับมา (ย้อนข้อ 3) — สะอาดสุด เพราะเป็น commit เดี่ยวที่แก้แค่ป้าย 3 ไฟล์: 
    git show ba3c2dd -- frontend/src/pages/admin/WalletAdjust.tsx \
  frontend/src/pages/admin/WalletTransfer.tsx \
  frontend/src/pages/admin/DepartmentAdjust.tsx | git apply -R
  4. ย้อน pagination (ข้อ 4) — ทำได้ (กลับไป Prev/Next + 20 แถว/หน้า + ตัดคอลัมน์ No.) แต่เป็นของที่ user แก้เองเมื่อ 30 มิ.ย. — ควรยืนยันก่อนว่าจะเอาออกจริง
ห้ามเอาไฟล์เก่าทับทั้งไฟล์
