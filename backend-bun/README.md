# ISB Backend вҖ” Bun + Elysia (new)

The new backend, written in TypeScript on Bun. Built to gradually replace the
Python FastAPI service in `../backend/` via the **strangler pattern** вҖ” both
backends run side-by-side, traffic is migrated one endpoint at a time, and the
legacy backend is deleted only after every route has moved over.

## Stack

| Concern        | Tool                                         |
| -------------- | -------------------------------------------- |
| Runtime        | Bun (>= 1.3)                                 |
| HTTP framework | Elysia                                       |
| ORM            | Drizzle ORM                                  |
| DB driver      | postgres-js                                  |
| Validation     | TypeBox (built into Elysia)                  |
| Auth           | `@elysiajs/jwt` (HS256, shared secret w/ FastAPI) |
| Tests          | `bun:test`                                   |
| Schema source  | `drizzle-kit introspect` against Postgres    |

## Quick start

```bash
# 1. Copy env template
cp .env.example .env
# Edit .env: paste DATABASE_URL (Railway DATABASE_PUBLIC_URL) and JWT_SECRET
# JWT_SECRET MUST match FastAPI's SECRET_KEY so tokens issued by either side
# can be verified by the other.

# 2. Install deps (from repo root вҖ” Bun workspace)
cd ..
bun install

# 3. Generate Drizzle schema from existing Postgres
cd backend-bun
bun run db:introspect

# 4. Run dev server
bun run dev
# вҶ’ http://localhost:3001/health
# вҶ’ http://localhost:3001/docs (OpenAPI / Swagger UI)

# 5. Run tests
bun test
```

## Environment

| Var            | Required | Description                                          |
| -------------- | -------- | ---------------------------------------------------- |
| `DATABASE_URL` | yes      | Postgres connection string (use `DATABASE_PUBLIC_URL` from Railway) |
| `JWT_SECRET`   | yes      | Must match `SECRET_KEY` from FastAPI `app/core/config.py` |
| `PORT`         | no       | Default `3001`                                       |
| `CORS_ORIGINS` | no       | Comma-separated. Default permissive.                 |
| `NODE_ENV`     | no       | `development` / `production`                         |

Tokens are HS256 with the same `SECRET_KEY` on both sides, so a token minted by
FastAPI is accepted by Elysia and vice versa.

## Conventions

- One route file per resource, mounted by `index.ts`.
- Business logic lives in `services/`; routes should be thin.
- All env access goes through `lib/config.ts`. No `process.env` elsewhere.
- All non-trivial endpoints require `requireAuth`; protect at the router level.
- New tables are migrated via `drizzle-kit generate` + `migrate`. Until cutover,
  schema is owned by Alembic in `../backend/alembic/` вҖ” do **not** push schema
  changes from Drizzle until FastAPI is retired.
