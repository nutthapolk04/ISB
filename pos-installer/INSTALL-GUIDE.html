ISB POS — คู่มือติดตั้งเครื่อง POS (ISB-POS-Setup)

คู่มือใช้ตัวติดตั้ง ISB-POS-Setup-1.0.0.exe เพื่อ provision เครื่อง POS Windows ให้พร้อมใช้ EDC + RFID ในครั้งเดียว

สารบัญ
  1. ภาพรวม
  2. สิ่งที่ต้องเตรียม
  3. ขั้นตอนติดตั้ง
  4. ตรวจรับหลังติดตั้ง
  5. ข้อควรระวัง
  6. แก้ปัญหาเบื้องต้น
  7. อภิธานศัพท์

1ภาพรวม

pos-installer/dist/ISB-POS-Setup-1.0.0.exe คือตัวติดตั้ง (NSIS) สำหรับเครื่อง POS Windows — build ล่าสุดเมื่อ 2026-07-16 บนเครื่อง Mac ด้วย makensis เป็นรุ่น OFFLINE ขนาด 157MB (ฝัง node_modules มาแล้ว ไม่ต้องใช้อินเทอร์เน็ตตอนติดตั้ง) ติดตั้งลง C:\ISB และรันซ้ำได้ปลอดภัย (idempotent — รอบใหม่จะ stop service เดิม / ปิด paywire ที่ค้างอยู่ก่อนเขียนทับเอง)

ตัวติดตั้งมี 3 component (ติ๊กไว้ทั้งหมดโดย default):

Component ทำอะไร
1. EDC USB driver
(whql_Driver2020)		 copy driver ไป C:\ISB\driver\ แล้วเปิด DriverInstall.exe ซึ่งเป็น wizard ของ vendor ให้ผู้ติดตั้งกดตามขั้นตอนเอง (ไม่ silent)
2. Paywire EDC bridge copy paywire.exe ไป C:\ISB\paywire\, สร้าง shortcut "Paywire Bridge" ใน Startup (all users) + Desktop แล้วเปิดโปรแกรมให้ทันที — เป็น tray app เสิร์ฟ pos.local.bridge.schooney.tech:7331
3. RFID bridge service
(ACR1252 + NSSM)		 วาง Node.js portable v26.5.0 + nssm.exe แล้วรัน install-rfid-service.ps1 เพื่อลงทะเบียน Windows Service ชื่อ rfid-bridge (auto-start, auto-restart เมื่อ crash, log ที่ C:\ISB\rfid-bridge\logs\out.log + err.log, ฟังที่ ws://localhost:9001) — สคริปต์รายงานเป็น ภาษาไทยทีละขั้น จบด้วยสรุปผล (Service Running ✅ + Port 9001 ✅)

โครงสร้าง C:\ISB หลังติดตั้ง

C:\ISB\
├── driver\               ← ไฟล์ EDC USB driver (whql_Driver2020)
├── paywire\
│   └── paywire.exe       ← Paywire EDC bridge (tray app)
├── rfid-bridge\
│   ├── rfid-server.js
│   ├── node_modules\     ← ฝังมากับ installer (รุ่น OFFLINE)
│   └── logs\
│       ├── out.log
│       └── err.log
├── node\
│   └── node.exe          ← Node.js portable v26.5.0
├── nssm.exe              ← ตัวจัดการ Windows Service
├── install-rfid-service.ps1
├── README.txt
└── uninstall.exe

ถอนการติดตั้งได้ผ่าน Add/Remove Programs รายการชื่อ "ISB POS Components" (หรือรัน C:\ISB\uninstall.exe โดยตรง) — จะ stop + remove service, ลบ shortcut และลบ C:\ISB ทั้งโฟลเดอร์ แต่ EDC driver จะยังติดตั้งอยู่กับ Windows ตามปกติของ driver

2สิ่งที่ต้องเตรียม

  • เครื่อง POS ที่รัน Windows 10/11 x64
  • สิทธิ์ Administrator บนเครื่องนั้น (จำเป็น — ลงทะเบียน Windows Service ไม่ได้ถ้าไม่มี)
  • USB drive ที่มีไฟล์ ISB-POS-Setup-1.0.0.exe
  • เครื่อง EDC + สาย USB
  • เครื่องอ่านบัตร ACR1252U
รุ่นที่ build ปัจจุบันเป็นรุ่น OFFLINE — ไม่ต้องใช้อินเทอร์เน็ตระหว่างติดตั้ง (node_modules ถูกฝังมาในตัว exe แล้ว ตัวติดตั้งจะข้าม npm install ไปเลย)

3ขั้นตอนติดตั้ง

  1. Copy ตัวติดตั้งลงเครื่อง POS

    เสียบ USB drive แล้ว copy ISB-POS-Setup-1.0.0.exe ลงเครื่อง (เช่นวางบน Desktop)

  2. รันแบบ Administrator

    คลิกขวาที่ไฟล์ exe → Run as administrator

  3. UAC

    เมื่อ Windows เด้งหน้าต่าง UAC ถามสิทธิ์ ให้กด Yes

  4. หน้า Welcome

    กด Next ผ่านหน้า Welcome ของตัวติดตั้ง

  5. หน้า Components

    ตรวจว่า component ทั้ง 3 ตัวติ๊กครบ (ค่า default ติ๊กไว้ให้แล้ว) → Next → ปล่อยโฟลเดอร์ติดตั้งเป็น C:\ISB → กด Install

  6. Driver wizard ของ vendor

    ระหว่างติดตั้งจะมีหน้าต่าง wizard ของ vendor (DriverInstall.exe) เด้งขึ้นมาสำหรับ EDC USB driver — กดตามขั้นตอนจนจบ

  7. หน้าต่าง PowerShell ของ RFID service

    จะมีหน้าต่าง PowerShell สีดำเปิดขึ้นมา รายงานการติดตั้ง RFID service เป็นภาษาไทยทีละขั้น (ตรวจสิทธิ์ → ตรวจไฟล์ → SCardSvr → เครื่องอ่าน → dependencies → ลงทะเบียน service → ตรวจ port)

    อย่าปิดหน้าต่างนี้! ปล่อยให้รันจนจบเอง
  8. รอสรุปผล

    ตอนจบต้องเห็นสรุปผลในหน้าต่าง PowerShell:

    Service status : ✅ Running ('rfid-bridge')
Port 9001      : ✅ Listening (ws://localhost:9001)
  9. Finish

    กลับมาที่หน้าต่างตัวติดตั้ง กด Finish เป็นอันเสร็จ

4ตรวจรับหลังติดตั้ง

  1. มองหา icon ของ paywire ใน system tray (มุมขวาล่างของ Windows) — ต้องรันอยู่
  2. เปิด PowerShell แล้วเช็ค service: 
    Get-Service rfid-bridge
    สถานะต้องเป็น Running
  3. เปิด Chrome ไปที่ https://isb.schooney.tech → หน้า canteen แล้วทดสอบ:
    • RFID: แตะบัตรที่เครื่องอ่าน — แอปต้องค้นหาสมาชิกให้อัตโนมัติ
    • EDC: กดชำระเงิน → เลือกจ่ายด้วย EDC — ต้องเห็น pill สีเขียว ● EDC connected
    • กดปุ่ม QR CODE — ยอดเงิน + QR ต้องขึ้นบนจอเครื่อง EDC

5ข้อควรระวัง

  • ต้อง Run as administrator เสมอ — ไม่งั้นลงทะเบียน Windows Service ไม่ได้ (สคริปต์จะหยุดพร้อม error)
  • ห้ามปิดหน้าต่าง PowerShell ระหว่างติดตั้ง — ปล่อยให้รันจนเห็นสรุปผล
  • ห้ามลบ/ย้ายโฟลเดอร์ C:\ISB หลังติดตั้ง — ทั้ง service และ shortcut autostart ชี้ path นี้ตรง ๆ
  • รุ่น OFFLINE เท่านั้นที่ไม่ต้องมีเน็ต — ถ้าเป็นรุ่น ONLINE (build โดยไม่มี prebuilt-node_modules.zip) เครื่อง POS ต้องมีอินเทอร์เน็ต + ติดตั้ง Visual Studio Build Tools (C++ workload) ไว้ก่อน
  • prebuilt node_modules ต้องมาจากเครื่อง POS Windows เท่านั้น — build.sh บน Mac มี guard ตรวจว่า pcsclite.node เป็น Windows x64 DLL (PE32+) และจะปฏิเสธ zip ที่มาจาก Mac (Mach-O) อัตโนมัติ — และต้อง build ด้วย Node v26.x ให้ ABI ตรงกับ Node portable ที่ฝังมา (v26.5.0)
  • เสียบ ACR1252U ก่อนติดตั้งได้ยิ่งดี — แต่ไม่เสียบก็ติดตั้งได้ (installer แค่เตือน bridge จะ detect เครื่องอ่านเองเมื่อเสียบภายหลัง)
  • ติดตั้งซ้ำได้ปลอดภัย (idempotent) — รัน exe ซ้ำเพื่อซ่อมเครื่องที่ service พังได้เลย
  • ถอนการติดตั้งจะไม่ถอน EDC driver — driver ติดตั้งอยู่กับ Windows ตามปกติ
  • CSP ของเว็บ isb.schooney.tech อนุญาต ws://localhost:9001 และ pos.local.bridge.schooney.tech:7331 แล้ว (แก้เมื่อ 2026-07-16) — เครื่องใหม่ไม่ต้องตั้งค่าฝั่งเว็บเพิ่ม

6แก้ปัญหาเบื้องต้น

อาการ วิธีแก้
service rfid-bridge ไม่ Running ดู error log:
Get-Content C:\ISB\rfid-bridge\logs\err.log -Tail 50
port 9001 ไม่ตอบ restart service:
C:\ISB\nssm.exe restart rfid-bridge
แตะบัตรแล้วเงียบ เช็ค Device Manager หมวด Smart card readers ว่าเห็นเครื่องอ่าน และเช็คว่า service SCardSvr รันอยู่
EDC pill ขึ้นแดง (not connected) เช็คว่า paywire รันอยู่ใน system tray — ถ้าไม่อยู่ให้ดับเบิลคลิก Desktop shortcut "Paywire Bridge" เปิดใหม่ แล้วเช็ค error เพิ่มด้วย F12 → Console
อยากติดตั้งซ้ำทั้งชุด รัน ISB-POS-Setup-1.0.0.exe ใหม่ได้เลย (idempotent — จัดการ service/ไฟล์เดิมให้เอง)

7อภิธานศัพท์

คำศัพท์ที่ใช้ในระบบติดตั้งเครื่อง POS และความหมายโดยย่อ
ศัพท์ ความหมาย
Installer (NSIS)ตัวติดตั้ง Windows ที่สร้างด้วย NSIS (Nullsoft Scriptable Install System) — build จาก installer.nsi ด้วย makensis ได้ไฟล์ .exe เดียว
Componentหน่วยติดตั้งที่เลือกติ๊กได้ในหน้า Components ของตัวติดตั้ง (ชุดนี้มี 3 ตัว: driver, Paywire, RFID)
NSSMNon-Sucking Service Manager — โปรแกรมที่ห่อ process ธรรมดา (เช่น node.exe) ให้กลายเป็น Windows Service ได้
Windows Serviceโปรแกรมที่ Windows รันเบื้องหลังโดยไม่ต้อง login และจัดการ start/stop ผ่านระบบ service
Auto-start / Auto-restartตั้งให้ service เริ่มเองตอนเปิดเครื่อง และ NSSM สั่ง restart ให้เองเมื่อ process crash
Node.js portableNode.js แบบแตกโฟลเดอร์ใช้ได้เลยโดยไม่ต้องติดตั้งลงระบบ — ฝังมาที่ C:\ISB\node (v26.5.0) ไม่พึ่ง Node บน PATH
node_modulesโฟลเดอร์ dependencies ของโปรเจกต์ Node.js ที่ได้จาก npm install
Native modulemodule ของ Node ที่มีส่วนเป็น binary ต้อง compile ต่อ platform (เช่น @pokusew/pcsclite ของ RFID bridge)
PE32+ vs Mach-Oรูปแบบไฟล์ binary ของ Windows x64 vs Mac — native module ที่ build บน Mac (Mach-O) รันบน Windows ไม่ได้ จึง zip node_modules ข้ามเครื่องต่าง platform ไม่ได้
OFFLINE / ONLINE modeโหมด build ของ installer — OFFLINE ฝัง node_modules สำเร็จรูปมาเลย ส่วน ONLINE ให้เครื่อง POS รัน npm install เอง (ต้องมีเน็ต + Build Tools)
VS Build ToolsVisual Studio Build Tools (workload "Desktop development with C++") ที่ node-gyp ใช้ compile native module บน Windows — จำเป็นเฉพาะโหมด ONLINE
Tray appโปรแกรมที่รันเป็น icon ใน system tray (มุมขวาล่าง) — paywire.exe เป็นแบบนี้
UAC / Run as administratorกลไกขอสิทธิ์ผู้ดูแลระบบของ Windows — ตัวติดตั้งนี้ต้องได้สิทธิ์นี้เสมอ (RequestExecutionLevel admin)
SCardSvrWindows Smart Card service ที่ระบบ PC/SC ต้องใช้ — สคริปต์ติดตั้งตั้งเป็น Automatic + Start ให้อัตโนมัติ
PC/SCมาตรฐานการคุยกับ smart card reader บน PC ที่เครื่องอ่าน ACR1252U ใช้
ACR1252Uเครื่องอ่านบัตร RFID/NFC ของ ACS ที่ต่อ USB กับเครื่อง POS ใช้แตะบัตรสมาชิก
paywire.exePaywire EDC bridge (tray app, .NET 8) ที่เชื่อมหน้าเว็บ POS กับเครื่อง EDC ผ่าน pos.local.bridge.schooney.tech:7331
ws://localhost:9001WebSocket endpoint ของ RFID bridge ที่หน้าเว็บใช้รับ UID บัตรแบบ real-time
Idempotentคุณสมบัติรันซ้ำได้ผลเหมือนเดิมโดยไม่พัง — installer นี้รันซ้ำเพื่อซ่อม/อัปเดตเครื่องเดิมได้ปลอดภัย
UninstallerC:\ISB\uninstall.exe (รายการ "ISB POS Components" ใน Add/Remove Programs) ใช้ถอนทุกอย่างยกเว้น EDC driver
EDC driver (whql_Driver2020)USB driver ของ vendor สำหรับเครื่อง EDC — ติดตั้งผ่าน wizard ของ vendor และติดอยู่กับ Windows แม้ถอน ISB POS Components แล้ว
